"# Chapter 11: Bemi v4.0: Ultra-Bandwidth & Execution Zenith Architecture\n\n## 11.2 3D Stacked V-Cache v2.0\n\n### 11.2.1 Doubling the Vertical SRAM Budget\nTo prevent the scaled **72 virtual threads** of Bemi v4.0 from saturating the memory hierarchy, the 3D Stacked V-Cache is upgraded to version 2.0.\n- **Capacity:** Expanded from 128 MB to **256 MB** of vertically stacked SRAM.\n- **Interconnect:** Connected using an ultra-wide 512-bit Through-Silicon Via (TSV) bus.\n- **Capture Efficiency:** Raising cache capacity to 256 MB increases the L1/L2 miss capture rate from 60% to **80%**.\n\n### 11.2.2 Blended Memory Latency Under v4.0 Cache\nWith an 80% capture rate at the L4 level, the blended memory latency is calculated as:\n$$ \\text{Latency}_{\\text{blended}} = (\\text{HitRate}_{L4} \\times \\text{Latency}_{L4}) + ((1 - \\text{HitRate}_{L4}) \\times \\text{Latency}_{\\text{DRAM}}) $$\n$$ \\text{Latency}_{\\text{blended}} = (0.80 \\times 15) + (0.20 \\times 40) = 12.0 + 8.0 = 20.0 \\text{ cycles} $$\nThis reduces the blended access latency of memory misses to **20 cycles** (down from 25 cycles in Bemi v3.0).\n"
